VMware安装MacOS 10.14

2020年3月6日
| 阅读

苹果系统自成一个生态系统，没有软硬件的情况下，安装确实不易。苹果也不想浪费资源到不买硬件的人身上，设置了重重障碍。

安装VMware15.5

点击下一步多次，增强键盘需要重启，但功能更多，最后一步输入注册码激活

安装unlocker

这是民间的力量，GitHub有很多个，以前的unlocker删除了，mk-unlocker可以使用。

下载解压后，右键使用管理员打开win-install.bat

最好是开启全局代理，不然下载VMware tools很慢

下载系统镜像

用于下一步制作安装光盘镜像，前提得有个MacOS环境，如果没有，只能下载别人制作好的。

一般从AppStore中下载，但是，AppStore中一般仅提供最新的系统镜像，而且不一定是完整的。

可以先从搜索网上下载，一般是dmg格式的，是通过安装app中的工具导出的

双击该dmg，可以看到一个磁盘挂载了，桌面有磁盘的快捷方式，打开后，里面就有完整的安装app，下一步需要使用该app

制作安装光盘

黑苹果玩家长期摸索的

# 创建一个dmg
hdiutil create -o /tmp/Mojave -size 6500m -layout SPUD -fs HFS+J

# 挂载dmg
hdiutil attach /tmp/Mojave.dmg -noverify -mountpoint /Volumes/Mojave

# 写入dmg，app路径可以根据实际情况写
echo Y | sudo /Applications/Install\ macOS\ Mojave.app/Contents/Resources/createinstallmedia --volume /Volumes/Mojave

# 卸载dmg
hdiutil detach /Volumes/Install\ macOS\ Mojave

# dmg转cdr
hdiutil convert /tmp/Mojave.dmg -format UDTO -o ~/Desktop/Mojave

# 删除dmg
rm /tmp/Mojave.dmg

安装

打开VMware，创建一个虚拟机，可以选自定义，稍后安装系统，系统一定要选MacOS，版本也要对应

内存要选4G以上，CPU和物理核心数一样，硬盘设为一个文件会快些，大小设为80GB，光盘选择上一步制作好的cdr

修改虚拟机配置文件，添加smc.version = "0"，很多年都有这个问题。

到达安装界面，选择磁盘工具，格式化硬盘，然后点击右下角的网络图标断开网络

打开实用工具中的终端，输入date 101010102015，因为证书过期，需要将时间设置到2016年的情人节以前

点击安装系统，应该可以安装成功。

输入apple ID时可以先跳过

调整分辨率

安装后分辨率特别小，先关机去掉光盘，开机后点击菜单中的安装VMware Tools，安装时会打开设置以允许。

重启，按住Command+R，进入恢复模式，打开终端，输入csrutil disable，以关闭系统完整性保护。

关闭MacOS，设置的显示器中去掉拉伸模式，启动后，可以看到，已经全屏了。如果不能全屏，重新安装VMware Tools试试。

不得不说，壁纸真没有上一个版本的山川河流好看。

禁用内存页面修整

VMware 10以后默认虚拟机启动后，会在磁盘上生成一个内存的映射文件，整理内存，但弊端非常大。

如果不禁用会导致，磁盘占用长期100%，特别是机械硬盘，而且读写速度超级慢，虚拟机里操作巨慢，挂起恢复也是超级慢。
