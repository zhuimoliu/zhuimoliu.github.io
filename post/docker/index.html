Docker的安装与基本使用

2020年2月16日
| 阅读

docker的特点：

  • 环境配置一致

  • 一次配置，到处运行

  • 更快速的应用交付和部署

  • 更便捷的升级和扩缩容

  • 更简单的系统运维

  • 更高效的计算资源利用

安装配置

# step 1: 安装必要的一些系统工具
sudo yum install -y yum-utils device-mapper-persistent-data lvm2
# Step 2: 添加软件源信息
sudo yum-config-manager --add-repo https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
# Step 3: 更新并安装Docker-CE
sudo yum makecache fast
sudo yum -y install docker-ce
# Step 4: 开启Docker服务
sudo service docker start
# 配置加速镜像，/etc/docker/daemon.json
sudo mkdir -p /etc/docker
#{"registry-mirrors": ["https://reg-mirror.qiniu.com/", "https://docker.mirrors.ustc.edu.cn", "https://hub-mirror.c.163.com", "https://weqjh8dwxxx.mirror.aliyuncs.com"]}
sudo systemctl daemon-reload
sudo systemctl restart docker

docker info 查看加速是否生效

基本使用

docker version

docker info

docker –help

docker images -a（含中间层） -q –digests（摘要） –no-trunc

docker search

docker pull ImageName:Tag

docker rmi -f xxx/$(docker images -qa)

docker pull xxx

docker run [options] ImageName [command]

常用选项：-i交互，-t终端，-d后台，–name名称，-p端口（可加主机IP），-P随机端口

docker ps -a -l（最近） -n（最近n个） -q –no-trunc

ctrl+P+Q，不停止退出

docker start/stop/restart/kill/rm/pause/unpause/commit/import/export/load/save/build

save保存的是image，export导出的是container

create 创建但不启动容器，diff查看容器变化

events，从 docker 服务获取容器实时事件，history，显示历史

port，查看端口映射，tag，给image打标签，wait，显示退出状态码

docker rm -f $(docker ps -a -q) 或者 docker ps -a -q | xargs docker rm

docker容器后台运行，必须有一个前台进程，且需要一直挂起运行（top，tail -f），否则会自动退出（kill self，因为没事可做）

docker logs -f（跟随最新） -t（加入时间戳） –tail（数字显示条数） xxx

docker top xxx

docker inspect xxx

docker exec -it xxx /bin/bash

docker attach xxx

docker cp xxx:/path/to/file /path/to/file

镜像使用联合文件系统，一层层的只读文件，overlay文件系统，使用bootloader加载bootfs，启动kernel，然后加载rootfs

docker commit -m “message” -a “author” xxx ImageName:Tag

数据卷类似挂载，-v，ro只读，默认可读写，可能需要–privileged=true

VOLUME指令添加数据卷

数据卷容器，解决容器之间数据共享，–volumes-from，只要有容器使用就一直存在

网络模式，默认情况下使用名为docker0的bridge，也可以使用host模式

Dockerfile

FROM Image:Tag  
MAINTAINER Author <user@email>  
RUN 执行命令
EXPOSE 暴露的端口
WORKDIR 默认工作目录
USER 运行的用户
ENV 环境变量，使用$JAVA_HOME：JAVA_HOME /usr/local/java
ADD 添加文件并解压：xxx.tar.gz /usr/local/
COPY 添加文件
VOLUME 卷
CMD 可以多个，最后一个为run时调用的命令，前面的为构建时执行的命令
# 如果run时指定，则原最后一个CMD不会执行
ENTRYPOINT 启动命令 run时可追加参数
ONBUILD 子镜像构建时执行的命令

docker build -f xxx -t xxx .

docker-compose

配置文件名：docker-compose.yml

version: '3'
services:
  nginx:
    image: nginx:1.16.1
    container_name: nginx
    volumes:
      - /docker/conf/nginx/nginx.conf:/etc/nginx/nginx.conf
      - /docker/logs:/data/logs
      - /docker/www:/data/www
    ports:
      - "8080:80"
    links:
      - php-fpm:php7.2-fpm

  php-fpm:
    build: ./
    container_name: php-fpm
    volumes:
      - /docker/conf/php/php.ini:/usr/local/etc/php/php.ini
      - /docker/conf/php/php-fpm.conf:/usr/local/etc/php/php-fpm.conf
      - /docker/conf/php/supervisord.conf:/etc/supervisor/supervisord.conf
      - /docker/logs:/data/logs
      - /docker/www:/data/www
      - /nfs/www:/nfs/www
    expose:
      - "9000"
    entrypoint: supervisord -c /etc/supervisor/supervisord.conf

docker-swarm 可以创建跨主机的容器集群，一般配合docker-machine创建多个主机

更好的方案是使用kubernets
